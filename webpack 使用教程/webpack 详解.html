webpack 详解

  • At its core, webpack is a static module bundler for modern JavaScript applications.
  • 本质上，webpack 是一个现代 JavaScript 应用程序的静态模块打包工具。
  • 当 webpack 处理应用程序时，它会递归地构建一个依赖关系图(dependency graph)，其中包含应用程序需要的每个模块，然后将所有这些模块打包成一个或多个 bundle。

为什么要使用 webpack 打包工具

  • 在ES6 之前要进行模块化开发，就必需借助于其他的工具；并且在模块化完成项目后，还要处理模块间的各种依赖关系，并将其进行整合打包
  • 而 webpack 其中一个核心就是能够让我们进行模块化开发，并且处理模块间的依赖关系
  • 不仅仅是 JavsScript 文件，CSS,impages,json 文件等等在 webpack 中都可以被当做模块来处理
  • 这就是 webpack 模块化的概念

webpack 可以打包符合 CommonJS 和 ES6 模块化规范的 js 代码

  • webpack 之所以能将所有的 js 代码打包成模块，是因为这些 js 代码本身就是模块化了的
  • 遵循 CommonJS 和 ES6 模块化规范的代码，就像一串串锁链，由一个个暴露的接口连接起来
